# Contribuindo

Contribuições são bem-vindas! Siga os passos abaixo:

## Reportar Problemas

Abra uma issue descrevendo:
- O que você esperava que acontecesse
- O que realmente aconteceu
- A URL da API utilizada (se aplicável)

## Sugerir Melhorias

- Novos endpoints ou APIs que não estão cobertos
- Exemplos adicionais
- Correções na documentação

## Pull Requests

1. Fork o repositório
2. Crie uma branch: `git checkout -b minha-feature`
3. Faça as alterações
4. Commit: `git commit -m "feat: descrição clara do que foi feito"`
5. Push: `git push origin minha-feature`
6. Abra um Pull Request

### Convenções

- Commits seguem [Conventional Commits](https://www.conventionalcommits.org/)
  - `feat:` — nova funcionalidade
  - `fix:` — correção
  - `docs:` — documentação
  - `refactor:` — refatoração
- Arquivos de referência em `references/` seguem o formato markdown com exemplos de `curl`
